# Compression and Transformation

This section describes how raw experience becomes structured memory.

---

## 1. From Slice to Meaning

A primary slice is not memory.

It must be transformed into a structured representation.

The system answers:
→ what happened?

---

## 2. Event Typing

The system determines the type of event.

This is done through structural comparison with existing patterns.

Comparison dimensions:
- structure  
- parameters  
- causality  
- context  
- outcome  

Result:
- known type  
- new type  
- noise  

---

## 3. Transformation

Compression is not data reduction.

It is transformation:

raw experience → structured episode

---

## 4. Episode Structure

A compressed episode contains:
- event type  
- key structure  
- causality  
- outcome  

It removes:
- redundant raw data  
- irrelevant details  

---

## 5. Storage Modes

Two modes exist:

### Normal Mode
- only compressed episode is stored  

### Extended Mode
- compressed episode  
- + part of raw slice  

---

## 6. Function

Transformation allows:
- abstraction  
- reuse  
- comparison  
- integration into the network  

---

## Summary

Memory does not store experience.

It converts experience into structure.
