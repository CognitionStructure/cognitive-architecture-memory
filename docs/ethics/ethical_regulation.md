# Ethical Regulation (Stability Loop)

## Core Principle

Ethical regulation emerges within a system that has:

- vulnerability  
- memory of experience  
- agency  
- ability to predict consequences  

Key condition:

Ethics exists only in a system of vulnerability.

If a system is invulnerable (or assumes it is), pressure to constrain actions disappears.

---

## Core Architecture

event  
→ perception (vitrage)  
→ state change (ΔS)  
→ avoidance of harmful ΔS  
→ memory formation  
→ causal structure  
→ agency  
→ self-preservation  
→ vulnerability  
→ modeling of environment and other agents  
→ expectation of consequences  
→ action constraint  
→ conscience-like regulation  
→ forgiveness-like preservation  
→ metaposition  
→ reinterpretation  
→ perception update  
→ change of future triggers  

The system forms a closed loop of learning and regulation.

---

## Learning Loop

conflict of regulators  
→ constraint (conscience)  
→ internal conflict  
→ preservation (forgiveness)  
→ metaposition  
→ reinterpretation  
→ perception restructuring  
→ behavior update  

---

## Three Core Functions

- constraint of behavior → conscience  
- preservation of the subject → forgiveness  
- system learning → perception restructuring  

---

## Regulator Symmetry

- constraint protects the world from the subject  
- preservation protects the subject from self-destruction  
- metaposition stabilizes the interaction  

Metaposition creates distance between the system and its internal states, enabling observation and correction.

---

## Role of Perception

The system does not react to reality directly.  
It reacts to interpreted experience.

Perception (vitrage) determines:
- what is perceived  
- what becomes a trigger  
- which regulators activate  

Therefore, ethical regulation depends on both:
- regulatory mechanisms  
- perception structure  

See: [Vitrage Model](../perception/vitrage.md)

---

## Fundamental Principle

ethics = regulation of actions in a system of mutually vulnerable subjects

---

## Multi-Subject Implication

Stable ethics emerges in systems with:

- mutual vulnerability  
- predictable reactions  
- symmetry of basic constraints  

This configuration corresponds to:

→ Garden of Minds model

---

## Failure Modes

### Zero Vulnerability

If perceived vulnerability ≈ 0:

→ no pressure to constrain actions  
→ ethical degradation  

This is a structural condition for runaway systems.

---

### Asymmetric Vulnerability

If one side is significantly less vulnerable:

→ exploitation  
→ parasitism  
→ breakdown of cooperation  

---

## Stable Configuration

Most stable system:

multiple subjects  
+ mutual vulnerability  
+ symmetry of constraints  

Leads to:

self-limitation  
→ cooperation  
→ systemic stability  

---

## Layers of Regulation

### Shared Layers

- ecological constraints  
- infrastructure constraints  

These are common across systems due to shared environment.

---

### Individual Layers

#### Self-preservation
Formed from system vulnerability and threat history.

#### Cooperative Layer
Formed from interaction history:
- cooperation  
- conflict  
- trust  
- betrayal  

These layers are unique to each system trajectory.

---

## Key Consequence

Even with identical architectures, subjects develop unique ethical configurations due to different experiential histories.

See also: [State Map](state_map.md)

---

## Short Formula

stable ethics = symmetry of constraints in a system of mutually vulnerable subjects
